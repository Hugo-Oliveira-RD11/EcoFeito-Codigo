* Authentication
** Register User
   - Endpoint: =POST /api/auth/register=
   - Description: Register new user (Customer, Seller, or Admin)
   - Headers: None
   - Request Body:
     #+BEGIN_SRC json
{
  "email": "user@example.com",
  "password": "password123",
  "role": "Customer",
  "firstName": "João",
  "lastName": "Silva",
  "storeName": "Minha Loja",
  "description": "Descrição da loja"
}
     #+END_SRC
   - Success Response (200):
     #+BEGIN_SRC json
{
  "userId": "507f1f77bcf86cd799439011",
  "email": "user@example.com",
  "role": "Customer"
}
     #+END_SRC
   - Error Responses:
     - 400: ={"error": "Email already exists"}=
     - 400: ={"error": "Validation failed", "errors": {...}}=

** Login
   - Endpoint: =POST /api/auth/login=
   - Description: Authenticate user and return JWT token
   - Headers: None
   - Request Body:
     #+BEGIN_SRC json
{
  "email": "user@example.com",
  "password": "password123"
}
     #+END_SRC
   - Success Response (200):
     #+BEGIN_SRC json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "email": "user@example.com",
  "role": "Customer",
  "userId": "507f1f77bcf86cd799439011",
  "expiresAt": "2024-01-15T10:30:00Z"
}
     #+END_SRC
   - Error Responses:
     - 401: ={"error": "Invalid credentials"}=
     - 401: ={"error": "Account is deactivated"}=

** Logout
   - Endpoint: =POST /api/auth/logout=
   - Description: Invalidate user token (client-side cleanup)
   - Headers: =Authorization: Bearer {token}=
   - Success Response (200):
     #+BEGIN_SRC json
{
  "success": true,
  "message": "Logged out successfully"
}
     #+END_SRC

** Change Password
   - Endpoint: =PUT /api/auth/change-password=
   - Description: Change user password
   - Headers: =Authorization: Bearer {token}=
   - Request Body:

  "currentPassword": "oldpassword123",
  "newPassword": "newpassword456"
}

   - Success Response (200):

   - Endpoint: =PUT /api/products/{id}=

** Delete Product
   - Endpoint: =DELETE /api/products/{id}=
   - Description: Deactivate product
   - Headers: =Authorization: Bearer {token}=
   - Success Response (204): No content

** Manage User
   - Endpoint: =PUT /api/admin/users/{id}=
   - Description: Update user status or role
   - Headers: =Authorization: Bearer {token}= (Admin only)
   - Request Body:
     #+BEGIN_SRC json
{
  "isActive": false,
  "role": "Seller"
}
     #+END_SRC
   - Success Response (200): Updated user

* Error Responses
  All endpoints return errors in consistent format:

  - *400 Bad Request*:
    #+BEGIN_SRC json
{
  "error": "Validation failed",
  "errors": {
    "email": ["Email is required", "Invalid email format"]
  }
}
    #+END_SRC

  - *401 Unauthorized*:
    #+BEGIN_SRC json
{
  "error": "Invalid token"
}
    #+END_SRC

  - *403 Forbidden*:
    #+BEGIN_SRC json
{
  "error": "Insufficient permissions"
}
    #+END_SRC

  - *404 Not Found*:
    #+BEGIN_SRC json
{
  "error": "Resource not found"
}
    #+END_SRC

  - *500 Internal Server Error*:
    #+BEGIN_SRC json
{
  "error": "An unexpected error occurred"
}
    #+END_SRC

* Status Codes Summary
  - 200: Success
  - 201: Created successfully
  - 204: Success no content
  - 400: Bad request / validation error
  - 401: Unauthorized
  - 403: Forbidden
  - 404: Not found
  - 500: Internal server error

* Notes
  - All dates are in ISO 8601 format
  - All prices are in decimal format with 2 decimal places
  - Pagination uses 1-based indexing
  - JWT tokens expire in 24 hours
  - Image URLs should be absolute URLs
  - Currency: BRL (Brazilian Real)
